背景

真實共創專案，聚焦 Web3 金融防詐情境。使用者對 NFT 認知落差大，安全疑慮與理解成本成為採用關鍵限制，需在法規與技術條件下設計可落地體驗。

簡介

將研究洞察轉化為可執行的驗證流程與介面策略，導入 AI 協作與 Vibe Coding，加速「研究 → 原型 → 驗證」循環，快速驗證多重假設。

問題陳述

Current financial users trust OTP and Face ID, but show strong resistance toward NFT-based identity verification
How might we introduce NFT authentication without breaking existing trust structures?

目的

在既有金融驗證（OTP／Face ID／裝置綁定）基礎上，驗證 NFT 作為金融身份驗證的可行性，降低新技術不信任感，提升登入與驗證流程的安全感與信任感。

研究過程

• 進行使用者訪談與問卷調查，蒐集金融 App 使用習慣、安全感來源與客服偏好
• 採用 COM-B 行為模型分析 NFT 驗證的採用阻力
• 導入 RAG 於研究流程中，整合金融安全資料與訪談內容，輔助假設生成與洞察驗證，降低知識落差與推論偏誤。
• 結合AI建立多項信任與採用相關假設，並以Vibe Coding 快速製作原型與 A/B Test 驗證使用者反應
• 透過 A/B Test 驗證不同驗證組合、文案、動畫與回饋機制對信任感的影響

研究內容

• 使用者對 NFT 的接受度高度取決於既有金融信任基礎
• NFT + 傳統驗證的混合模式能顯著降低心理門檻
• 金錢型回饋與優惠誘因，比抽象安全說明更能促進採用
• 客服支援與清楚的安全處理流程，是建立信任的必要條件

成果與影響

• 驗證「混合驗證（NFT + 傳統）」較單一 NFT 驗證更具採用意願
• 登入後即呈現優惠與身份摘要，能有效提升使用者選擇 NFT 驗證
• 金錢型回饋顯著高於其他獎勵形式
• 清楚的法規與安全文案可進一步提升安心感
• 強化客服入口能補足 Web3 技術的不信任缺口
